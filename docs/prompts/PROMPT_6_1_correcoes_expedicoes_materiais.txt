# PROBLEMA 1 — TIMER DE EXPEDIÇÃO NÃO ESTÁ SEMPRE EM HH:MM:SS

No ExpeditionSystem, a função formatTimeRemaining pode retornar MM:SS quando horas = 0.

Mas a Bíblia e os prompts pedem padrão:

HH:MM:SS

Exemplos:

00:00:45
00:12:08
01:43:22

---

# CORREÇÃO OBRIGATÓRIA 1

Atualizar formatTimeRemaining em ExpeditionSystem para sempre retornar HH:MM:SS.

Usar o mesmo padrão do TimeManager, por exemplo:

const totalSeconds = Math.max(0, Math.floor(ms / 1000));
const hours = Math.floor(totalSeconds / 3600);
const minutes = Math.floor((totalSeconds % 3600) / 60);
const seconds = totalSeconds % 60;

return `${String(hours).padStart(2, '0')}:${String(minutes).padStart(2, '0')}:${String(seconds).padStart(2, '0')}`;

Critério:

- Se faltar menos de 1 minuto, mostrar 00:00:45.
- Se faltar 12 minutos, mostrar 00:12:00.
- Se faltar 1 hora, mostrar 01:00:00.
- Não mostrar só 12:00.

# PROBLEMA 2 — MATERIAIS DE SAVES ANTIGOS PODEM QUEBRAR DISPLAY

O projeto atual usa MaterialInventory como objeto:

materials = {
  living_ash: number,
  ancient_stone: number,
  shell_fragment: number,
  memory_echo: number
}

Mas o GameStateValidator aceita materiais como array para compatibilidade com saves antigos.

Isso é bom, mas MaterialDisplay e ExpeditionSystem podem receber um array antigo e tentar tratar como objeto de MaterialInventory.

Isso pode gerar erro se Object.entries retornar índices de array em vez de MaterialId.

---

# CORREÇÃO OBRIGATÓRIA 2

Criar uma função utilitária simples para normalizar materiais antigos.

Pode ser em um novo arquivo:

src/utils/materials.ts

ou em ExpeditionSystem/MaterialDisplay se preferir manter simples.

Exemplo conceitual:

export function createEmptyMaterialInventory(): MaterialInventory {
  return {
    living_ash: 0,
    ancient_stone: 0,
    shell_fragment: 0,
    memory_echo: 0,
  };
}

export function normalizeMaterialInventory(input: unknown): MaterialInventory {
  const empty = createEmptyMaterialInventory();

  if (!input || typeof input !== 'object' || Array.isArray(input)) {
    return empty;
  }

  const obj = input as Partial<Record<MaterialId, number>>;

  return {
    living_ash: typeof obj.living_ash === 'number' ? obj.living_ash : 0,
    ancient_stone: typeof obj.ancient_stone === 'number' ? obj.ancient_stone : 0,
    shell_fragment: typeof obj.shell_fragment === 'number' ? obj.shell_fragment : 0,
    memory_echo: typeof obj.memory_echo === 'number' ? obj.memory_echo : 0,
  };
}

Se quiser preservar array antigo com itens de material, pode mapear, mas não é obrigatório agora. O importante é não crashar.
