# FONTE DA VERDADE

O projeto deve obedecer à ARC: AETHON — Game Flow Bible v1.2.

Regras centrais:

- Uma conta = um ovo ou um dragão, nunca os dois.
- Depois do nascimento, o ovo deixa de existir.
- A DragonScreen é a tela principal quando hasDragon = true.
- O dragão é único e deve criar vínculo com o jogador.
- Alimentação deve aumentar vitalidade e influenciar traços.
- Diário deve registrar memórias narrativas, não logs técnicos.
- Expedições, ninho e profissões existem, mas NÃO devem ser implementados completamente neste prompt.
- Limiar e Convergência continuam apenas como estrutura/taxonomia.

Transformar a DragonScreen de placeholder em uma tela funcional inicial com:

# TELA DO DRAGÃO

A DragonScreen deve mostrar, de forma clara:

- nome do dragão;
- tipo do dragão;
- elemento dominante;
- vitalidade;
- traços de personalidade;
- cristais;
- diário;
- seção de alimentação;
- seção bloqueada/placeholder de expedições futuras.

Se houver função getDragonTypeById em dragonTaxonomy.ts, usar para exibir:

- nome da linhagem;
- categoria;
- elementos;
- descrição/lore curta, se existir.

Se o tipo não for encontrado, usar fallback visual seguro:
"Dragão Desconhecido"

Mas não deve quebrar o app.

---

# VITALIDADE

A vitalidade deve ser um número entre 0 e 1.

Exibir como porcentagem:

vitality * 100

Exemplo:

Vitalidade: 85%

Se alimentar, aumentar vitalidade, com máximo 1.

Exemplo:

newVitality = Math.min(1, currentVitality + FOOD_VITALITY_GAIN)

Criar constante se ainda não existir:

FOOD_VITALITY_GAIN = 0.15

Ou usar valor semelhante já existente no projeto.

---

# CRISTAIS

O MVP deve trabalhar com cristais elementais.

Se o projeto já usa um formato de crystals, respeitar o formato existente.

Preferência de estrutura:

crystals = {
  fire: number,
  water: number,
  earth: number,
  air?: number,
  metal?: number
}

Se o projeto usa number simples, adaptar com cuidado sem quebrar tipos.

Para este prompt, a alimentação usa:

- Cristal de Fogo
- Cristal de Água
- Cristal de Terra

Se ainda não houver cristais suficientes para testar, o DebugPanel DEV ONLY pode ter uma opção de adicionar cristais de teste, desde que seja claramente DEV ONLY e não pareça recurso do jogo.

Não criar compra de cristais.
Não criar monetização.

# ALIMENTAÇÃO — REGRAS

Ao clicar em uma comida:

1. Verificar se o dragão existe.
2. Verificar se há cristal suficiente.
3. Se não houver cristal:
   - não alimentar;
   - mostrar mensagem:
     "Você não tem cristais suficientes para preparar esta comida."
4. Se houver:
   - consumir 1 cristal do elemento correto;
   - aumentar vitalidade;
   - aplicar tendência/traço;
   - salvar;
   - mostrar mensagem narrativa da comida;
   - opcionalmente criar entrada curta no diário.

Pode ser:

- card expansível;
- lista simples;
- modal simples.

O diário deve exibir:

- texto;
- data ou dia, se disponível;
- categoria/tipo, se disponível.

Não registrar toda ação de forma repetitiva demais.

Para este prompt, pode criar entrada de diário apenas quando:

- é a primeira vez que o dragão come uma comida específica;
ou
- com chance simples;
ou
- sempre, se for mais simples, mas com texto narrativo.

Sugestão simples:
