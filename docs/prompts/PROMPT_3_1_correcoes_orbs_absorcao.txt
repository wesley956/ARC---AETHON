# CORREÇÕES OBRIGATÓRIAS

## 1. Corrigir pointercancel no DraggableOrb

Atualmente, quando ocorre pointercancel, o componente limpa apenas o estado local do orb.

Isso pode deixar a EggScreen achando que ainda está arrastando.

Corrigir criando uma prop, por exemplo:

onDragCancel?: () => void

ou usando uma função equivalente.

Quando pointercancel acontecer:

- limpar estado local;
- liberar pointer capture com segurança;
- chamar onDragCancel;
- a EggScreen deve executar:
  - setIsDragging(false);
  - setIsOverEgg(false);
  - remover estado de drag visual;
  - permitir scroll novamente.

Critério:
Se o drag for cancelado por mobile/browser, a tela não pode ficar travada em modo dragging.

---

## 2. Proteger releasePointerCapture

Antes de chamar releasePointerCapture, verificar se é seguro.

Usar algo como:

if (target.hasPointerCapture(e.pointerId)) {
  target.releasePointerCapture(e.pointerId);
}

Critério:
Não pode gerar erro caso o elemento não tenha mais captura do pointer.

---

## 3. Aumentar área de drop do ovo

Hoje a área de drop usa raio pequeno demais para mobile.

Aumentar a área de detecção para aproximadamente:

const EGG_DROP_RADIUS = 110;

Criar constante local ou em gameConstants.ts.

Critério:
Soltar o orb perto do ovo deve funcionar de forma confortável em mobile.

---

## 4. Padronizar timer para HH:MM:SS

Atualizar formatTimeRemaining para mostrar:

HH:MM:SS

Exemplos:

01:43:22
00:12:08
00:00:45

Se o timer estiver pronto:

Pronto!

Critério:
O texto da EggScreen deve ficar:

Próximo orb em: 01:43:22

## 5. Mostrar “Tray cheio” quando o limite de orbs for atingido

Se:

egg.availableOrbs.length >= ORB_TRAY_MAX

Então a tela não deve mostrar “Pronto!” como se houvesse orb para gerar.

Mostrar:

Tray cheio

ou:

Limite de orbs atingido

Critério:
Quando o tray estiver com 8/8 orbs, o jogador entende que precisa usar orbs antes de receber mais.

---

## 6. Limpar checagem duplicada de nascimento

Hoje a transição para HatchScene funciona pelo useEffect que observa:

egg.maturationProgress >= MATURATION_HATCH_THRESHOLD

Mantenha essa lógica.

Mas dentro de completeAbsorption, evite depender de save antigo do closure para decidir nascimento.

Pode fazer de uma destas formas:

Opção A:
- Remover a checagem manual com setTimeout;
- Deixar apenas o useEffect navegar quando a maturação atualizar.

Opção B:
- Calcular newMaturation dentro do updater;
- usar uma flag segura local para navegar depois.

Preferência:
Use a Opção A se for mais simples.

Critério:
Quando maturação chegar a 100%, o jogador ainda deve ir para HatchScene normalmente.
