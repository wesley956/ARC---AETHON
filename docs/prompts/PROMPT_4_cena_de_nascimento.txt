O projeto deve obedecer à ARC: AETHON — Game Flow Bible v1.2.

Regras centrais:

- Uma conta = um ovo ou um dragão, nunca os dois.
- O nascimento é irreversível.
- Ao nascer, o ovo deixa de existir.
- O nascimento deve criar dragonData.
- O nascimento deve apagar eggData.
- O jogador deve nomear o dragão.
- O diário deve receber a entrada do Dia 1.

- No MVP local, Limiar pode existir apenas como estrutura/tipo previsto, mas não deve ser garantido por Vazio alto.
- Dragão da Convergência é único do criador e deve ficar apenas previsto na estrutura, não jogável agora.

# REGRAS DE RESOLUÇÃO DO TIPO

IMPORTANTE:
Não transformar Vazio alto em garantia automática de Limiar.

No MVP local, use esta ordem:

1. Calcular o total de energia pública:

fireEnergy + waterEnergy + earthEnergy + airEnergy + metalEnergy

2. Se o total público for 0, usar fallback seguro:
- tipo: earth_pure ou equivalente de Terra
- elemento dominante: earth
- motivo: evitar erro técnico

3. Calcular proporções públicas:

fireRatio
waterRatio
earthRatio
airRatio
metalRatio

4. Identificar o elemento público dominante.

5. Se algum elemento público tiver proporção >= 0.60:
- retornar dragão puro correspondente.

6. Se nenhum elemento público tiver >= 0.60:
- pegar os dois maiores elementos públicos.
- retornar dragão fundido correspondente.

7. Sobre Limiar:
- Se voidEnergy for alto, apenas marcar um campo narrativo interno como:
  voidTouched: true
- NÃO transformar automaticamente em Dragão do Limiar.
- NÃO entregar Dragão do Limiar por regra local.
- O Limiar real será implementado futuramente com servidor e controle global.

8. Sobre Convergência:
- Não resolver Convergência para jogadores comuns.
- Deixar apenas estrutura/tipo previsto na taxonomia.
- Não criar Dragão da Convergência neste prompt.

# SEQUÊNCIA VISUAL DA HATCHSCREEN

Implementar uma sequência simples, bonita e funcional.

Não precisa ser cinema perfeito.
Precisa ser emocional e estável.

Fases sugeridas:

1. PREPARING
   - Carrega eggData.
   - Resolve tipo do dragão.
   - Mostra ovo centralizado.
   - Texto: "Algo se move dentro da casca..."

2. CRACKING
   - Ovo treme.
   - Rachaduras aparecem visualmente.
   - Luz do elemento dominante começa a vazar.

3. PULSE
   - Ovo pulsa com glow elemental.
   - Texto curto: "A energia acumulada responde."

4. BURST
   - Efeito de brilho/partículas simples.
   - Ovo desaparece ou abre.
   - Tela pisca levemente.

5. REVEAL
   - Mostrar representação visual do dragão.
   - Pode ser uma silhueta/ícone/arte simbólica por enquanto.
   - Mostrar nome do tipo resolvido.
   - Mostrar frase narrativa.

6. NAMING
   - Input para o jogador nomear o dragão.
   - Botão "Confirmar nome".

7. COMPLETE
   - Criar dragonData.
   - Atualizar save.
   - Navegar para DragonScreen.

---

# CONTROLE DA SEQUÊNCIA

A sequência pode ser automática com timers simples.

Exemplo de duração:

- PREPARING: 1000ms
- CRACKING: 1800ms
- PULSE: 1800ms
- BURST: 1200ms
- REVEAL: permanece até jogador avançar ou mostra botão
- NAMING: aguarda input do jogador

Pode haver botão "Continuar" após revelar o tipo para ir para nomeação.

Não deixar a cena travada se timer falhar.
