# PROBLEMA 1 — DRAGÃO NASCE COM VITALIDADE 100%

Hoje o DragonFactory cria o dragão com vitalidade cheia:

vitality: 1.0

Isso faz com que, logo após o nascimento, o jogador veja as comidas mas não consiga alimentar, porque o dragão já está satisfeito.

Isso atrapalha o primeiro loop de vínculo.

---

# CORREÇÃO OBRIGATÓRIA 1

Alterar a vitalidade inicial do dragão criado no nascimento.

Usar:

vitality: 0.75

ou uma constante:

INITIAL_DRAGON_VITALITY = 0.75

Critério:

- Depois de nascer, o dragão deve poder comer pelo menos uma comida.
- A vitalidade continua entre 0 e 1.
- Não alterar saves já existentes desnecessariamente.

---

# PROBLEMA 2 — SAVES ANTIGOS PODEM NÃO TER foodsEatenFirstTime

No Prompt 5 foi adicionado o campo:

foodsEatenFirstTime: string[]

Mas saves criados antes disso podem não ter esse campo.

Se feedDragon usar:

dragonData.foodsEatenFirstTime.includes(foodId)

e o campo estiver undefined, pode quebrar.

---

# CORREÇÃO OBRIGATÓRIA 2

Tornar o uso de foodsEatenFirstTime seguro.

Em qualquer lugar que usar esse campo, aplicar fallback:

const foodsEatenFirstTime = dragonData.foodsEatenFirstTime ?? [];

Ao salvar depois de alimentar, garantir que o campo passe a existir:

foodsEatenFirstTime: updatedFoodsEatenFirstTime

Critério:

- Save antigo sem foodsEatenFirstTime não pode quebrar a alimentação.
- Depois de alimentar, o save deve passar a ter foodsEatenFirstTime como array.
- Não resetar save antigo.
- Não recriar dragonData do zero.

---

# CORREÇÃO OBRIGATÓRIA 3 — VALIDAÇÃO / NORMALIZAÇÃO LEVE

Se existir GameStateValidator ou função de normalização de save, ajustar de forma segura para aceitar saves antigos.

Opção preferida:

- Não marcar save antigo como inválido só porque foodsEatenFirstTime está ausente.
- Se possível, normalizar para [] em algum ponto seguro.
- Se o projeto não tem sistema de migração, apenas garantir fallback em DragonCareSystem e DragonScreen já é suficiente.

Não criar sistema complexo de migração agora.

Critério:

- Saves antigos continuam abrindo.
- Saves novos ficam com foodsEatenFirstTime definido.
