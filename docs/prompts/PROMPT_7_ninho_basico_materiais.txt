# DADOS DO NINHO

Se já existir nestData no DragonData, usar e expandir com cuidado.

Se não existir ou estiver incompleto, criar estrutura segura.

Sugestão de estrutura:

nestData = {
  comfort: number,
  style: 'basic' | 'warm' | 'stone' | 'memory',
  slots: {
    base: NestSlot | null,
    comfort: NestSlot | null,
    relic: NestSlot | null,
  },
  appliedUpgrades: string[],
  lastUpdatedAt: string | null
}

NestSlot sugerido:

{
  id: string,
  name: string,
  materialId: string,
  slotType: 'base' | 'comfort' | 'relic',
  comfortBonus: number,
  elementalAffinity?: 'fire' | 'water' | 'earth' | 'air' | 'metal' | 'void',
  description: string
}

Se o projeto já tem tipos diferentes, adaptar ao padrão existente.

IMPORTANTE:
Não quebrar saves antigos.
Usar fallback seguro se nestData estiver ausente.

Exemplo:

const nestData = dragonData.nestData ?? createDefaultNestData();

# APLICAR UPGRADE DO NINHO

Ao clicar em um upgrade:

1. Verificar se dragonData existe.
2. Normalizar materials usando as funções já existentes em src/utils/materials.ts.
3. Verificar se há material suficiente.
4. Se não houver:
   - não aplicar;
   - mostrar feedback:
     "Você não tem material suficiente para alterar o ninho."
5. Se houver:
   - consumir material;
   - aplicar upgrade ao slot correto;
   - atualizar comfort;
   - atualizar style;
   - adicionar upgrade em appliedUpgrades;
   - salvar;
   - criar entrada narrativa no diário;
   - mostrar feedback visual.

IMPORTANTE:
Se o slot já tiver upgrade aplicado, não duplicar efeito infinitamente.

Opções aceitáveis:

Opção A:
- Permitir substituir o slot, mas recalcular conforto a partir dos slots ativos.

Opção B:
- Bloquear reaplicar o mesmo upgrade e mostrar:
  "Esse ajuste já faz parte do ninho."

Preferência:
Use a opção mais simples e segura.

Critério:
O jogador não pode farmar conforto infinito aplicando o mesmo upgrade repetidamente.

# CONFORTO DO NINHO

Comfort deve ser seguro.

Sugestão:

- começa em 0;
- máximo 100;
- cada upgrade aumenta um pouco;
- mostrar como porcentagem ou barra.

Exemplo:

comfort = Math.min(100, soma dos comfortBonus dos slots aplicados)

Preferência:
Recalcular conforto a partir dos slots ativos para evitar duplicação/bug.

Não implementar bônus complexos ainda.
