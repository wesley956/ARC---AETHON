# ZONA MVP

Implementar a zona:

## Ruínas de Ignareth

Descrição:
"As bordas queimadas de uma civilização antiga. Mesmo depois de séculos, algumas pedras ainda guardam calor."

Camadas:

## Camada 1 — Fronteira

Duração:
- Para produção: 30min a 2h.
- Para desenvolvimento/teste, pode usar duração reduzida se existir modo DEV ONLY.
- Se usar duração reduzida, deixar claro que é DEV ONLY.

Risco:
- Nenhum.

Recompensas:
- 1 a 3 Cristais de Fogo.
- 1 material comum de ruína.
- Chance baixa de fragmento narrativo.

Entrada de diário:
"Dia [X] — Fui às bordas de Ignareth hoje. O cheiro de cinzas não me incomodou."

## Camada 2 — Interior

Duração:
- Para produção: 3h a 6h.
- Para desenvolvimento/teste, pode usar duração reduzida se existir modo DEV ONLY.
- Se usar duração reduzida, deixar claro que é DEV ONLY.

Risco:
- 10% chance de lesão leve.

Recompensas:
- 2 a 5 cristais mistos.
- 1 a 2 materiais incomuns.
- Chance de Eco de Memória.

Entrada de diário normal:
"Dia [X] — Fui mais fundo dessa vez. Vi algo que não consigo descrever ainda."

Entrada de diário em caso de lesão:
"Dia [X] — Voltei machucado. Não vou contar o que encontrei lá."

# FLUXO DE EXPEDIÇÃO

## Enviar expedição

Ao clicar em "Enviar em expedição":

1. Verificar se dragonData existe.
2. Verificar se isOnExpedition é false.
3. Verificar se isInjured é false.
4. Escolher camada.
5. Calcular duração.
6. Definir:
   - isOnExpedition = true
   - expeditionStartTime = agora
   - expeditionReturnTime = agora + duração
   - expeditionLayer = camada escolhida
7. Salvar.
8. Mostrar feedback:
   "[Nome do dragão] partiu para as Ruínas de Ignareth."

## Durante expedição

DragonScreen deve mostrar:

- status: "Em expedição"
- zona: Ruínas de Ignareth
- camada
- tempo restante em HH:MM:SS
- texto narrativo curto:
  "Ele está longe, entre cinzas antigas."

Não permitir alimentar? 
Pode permitir alimentar apenas se isso não complicar, mas recomendação:
- Enquanto em expedição, bloquear alimentação com mensagem:
  "Ele está em expedição agora."

Não permitir iniciar outra expedição.

## Retorno

Quando agora >= expeditionReturnTime:

- Mostrar botão:
  "Coletar retorno"

Ao clicar:

1. Calcular recompensa pela camada.
2. Aplicar cristais.
3. Aplicar materiais.
4. Calcular lesão se camada 2.
5. Criar entrada de diário.
6. Definir:
   - isOnExpedition = false
   - expeditionStartTime = null
   - expeditionReturnTime = null
   - expeditionLayer = null
7. Se houve lesão:
   - isInjured = true
   - recoveryUntil = agora + duração de recuperação
8. Salvar.
9. Mostrar feedback:
   "[Nome do dragão] voltou da expedição."

# LESÃO LEVE

Apenas Camada 2 tem risco.

Chance:

10%

Se ocorrer:

- isInjured = true
- recoveryUntil = agora + tempo de recuperação

Tempo de recuperação sugerido:

- Produção: 1h
- DEV ONLY: pode ser menor para teste

Enquanto machucado:

- não pode iniciar expedição;
- mostrar status:
  "Machucado"
- mostrar tempo de recuperação, se houver;
- pode continuar vendo diário e recursos;
- alimentação pode continuar, se já existir, e pode ajudar no futuro, mas não precisa acelerar recuperação neste prompt.

Quando recoveryUntil passar:

- ao abrir o app ou atualizar DragonScreen, limpar:
  - isInjured = false
  - recoveryUntil = null

Não criar tratamento complexo ainda.
Não criar item de cura.
Não criar monetização para curar.

---

# TIME/OFFLINE

O sistema deve funcionar mesmo se o app for fechado.

Ao abrir o app:

- se o dragão está em expedição e expeditionReturnTime já passou, mostrar retorno disponível.
- não coletar automaticamente as recompensas sem o jogador clicar.
- o jogador deve clicar em "Coletar retorno" para receber recompensas e diário.

Se recoveryUntil já passou:

- pode limpar lesão automaticamente de forma segura.

Você pode implementar isso em:

- TimeManager
- DragonScreen
- ExpeditionSystem

Escolha a forma mais simples, robusta e compatível com a arquitetura atual.
